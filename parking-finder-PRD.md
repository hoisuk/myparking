# 주차 위치 기록 앱 (내 차 어디있지?) — PRD

| 항목 | 내용 |
|---|---|
| 문서 버전 | v1.0 |
| 작성일 | 2026-07-11 |
| 대상 | 개인/가정용 (지하 3층 주차장 보유 세대) |
| 플랫폼 | 모바일 웹 (홈 화면 추가 방식), 단일 HTML 파일 |

---

## 1. 배경 및 문제 정의

거주 중인 건물의 주차장은 **지하 1층(B1)부터 지하 3층(B3)**까지 있음. 매번 다른 층에 주차하다 보니, 다음날 또는 며칠 뒤 차를 찾을 때 **어느 층에 주차했는지 기억이 나지 않는 문제**가 반복적으로 발생함.

기존에는 별도 메모 앱이나 사진첩에 임시로 기록했으나, 층수·구역·시간 정보가 흩어져 있어 확인이 번거로움. 주차 직후 몇 번의 탭만으로 위치를 기록하고, 다음에 열었을 때 바로 확인할 수 있는 전용 도구가 필요함.

## 2. 목표

- 주차 직후 **단 한 번의 탭**으로 층수를 기록한다.
- 다음에 앱을 열었을 때 **현재 저장된 위치를 한눈에** 확인할 수 있다.
- 차를 찾은 뒤에는 위치를 지워 다음 주차를 위한 빈 상태로 되돌린다.

### 목표가 아닌 것 (Non-goals)

- 과거 주차 이력(히스토리) 기록 — 사용자 요청으로 제외됨. 현재 위치 1건만 관리.
- 여러 명(가족 등)이 공유하는 멀티유저 기능 — 현재는 개인 기록 전용.
- 실제 주차장 지도/내비게이션 연동.

## 3. 타겟 사용자

- 지하 다층 주차장을 이용하는 1인/가정 사용자 (본인 및 가족 구성원).
- 모바일 브라우저(Safari/Chrome)에서 홈 화면에 추가해 앱처럼 사용.

## 4. 핵심 사용자 시나리오

1. 지하주차장에 차를 세운다.
2. 앱을 열고 주차한 층(B1/B2/B3)을 탭한다 → 선택된 버튼이 반짝이며(펄스 애니메이션) 잠시 후 자동 저장된다.
3. 다음날, 앱을 열면 "B2, 오늘 오후 3:24 저장" 형태로 즉시 확인.
4. 차를 찾은 후 "차 찾음 · 위치 지우기"를 눌러 초기화.

## 5. 기능 요구사항

### 5.1 층 선택 및 저장

| 기능 | 설명 |
|---|---|
| 층 선택 버튼 | B1 / B2 / B3, 3개의 큰 버튼으로 제공 |
| 선택 상태 표시 | 선택된 층 버튼에 색상 강조 + 은은한 펄스(반짝임) 애니메이션으로 명확히 표시 |
| 원탭 자동 저장 | 층 버튼을 탭하면 펄스 애니메이션이 잠깐 재생된 뒤 층수 + 저장 시각이 자동 저장됨 (추가 확인 단계 없음) |
| 층별 색상 코드 | B1(골드) / B2(청록) / B3(코랄)로 시각적으로 구분, 실제 주차장 층별 안내 색상처럼 UI 전반에 반영 |

### 5.2 현재 위치 상태 카드

- 저장된 층(대형 숫자), 저장 시각을 한 화면에 표시.
- "차 찾음 · 위치 지우기" 버튼으로 현재 저장값 삭제.

### 5.3 데이터 저장

- `window.storage` (개인 저장소, `shared=false`)를 사용해 앱을 껐다 켜도 데이터가 유지됨.
- 저장 키: 현재 위치 정보 1건(`current`)만 관리. 과거 이력은 저장하지 않음.

### 5.4 모바일 홈 화면 등록

- 브라우저 자체 "홈 화면에 추가" 기능을 통해 아이콘처럼 실행 가능하도록 반응형 레이아웃 제공.

## 6. 비기능 요구사항

- **단일 HTML 파일**로 구성 (Vanilla JS, 별도 백엔드 없음).
- 반응형 디자인: 모바일 화면(최대 480px 기준) 최적화, 큰 탭 타겟.
- 개인정보: 저장되는 층수/시간 정보는 개인 저장소에만 남고 공유되지 않음.

## 7. 기술 스택

| 영역 | 내용 |
|---|---|
| 프론트엔드 | HTML / CSS / Vanilla JavaScript (프레임워크 없음) |
| 데이터 저장 | Claude 아티팩트 `window.storage` API (개인 저장소) |
| 폰트 | Oswald(층수 숫자), Noto Sans KR(본문), JetBrains Mono(시각 표시) |

## 8. 제약 사항 및 알려진 이슈

- 이 앱은 **Claude 아티팩트 실행 환경 내에서 열릴 때** `window.storage`가 정상 동작함. 외부에 별도로 호스팅(예: 개인 서버)하면 저장 기능이 동작하지 않을 수 있음.
- 공개 링크로 게시(Publish)하려면 Claude 설정의 **Capabilities > Code execution and file creation**이 켜져 있어야 함.
- 한 번 Unpublish하면 동일 링크로 재게시가 불가능하며, 새 아티팩트를 만들어야 함.

## 9. 향후 고려 사항 (Out of scope, 추후 논의)

- 가족 구성원과 위치를 공유하는 멀티유저 모드 (shared storage 활용).
- 주차 시간 제한이 있는 경우, 일정 시간 후 알림.
- 구역/번호 자유 검색 및 자동완성.
- 사진 촬영 + AI 텍스트 추출(OCR) 기능 — v1.0에서 제외되었으나 추후 재검토 가능.
